## Demo Overview

The WordPress Feature API Demo is a fully functional example plugin that demonstrates how to implement, register, and use the WordPress Features API in a real-world context. It showcases the API's capabilities through a practical implementation that includes both server-side and client-side features, a conversational interface, and tooling that demonstrates the extensibility of the system.

## Key Concepts Illustrated

The demo illustrates several core concepts of the WordPress Feature API:

1. **Feature Registration** - Examples of registering both resource and tool features
2. **Server-Client Communication** - Demonstrating how server and client can exchange capabilities 
3. **Feature Discovery** - How features can be discovered and executed programmatically
4. **Schema Definition** - Using JSON Schema to define inputs and outputs
5. **Permission Control** - Implementing permission callbacks for features
6. **Eligibility Checks** - Using conditional checks to determine if a feature should be available
7. **Conversational Interface** - A practical implementation showing how AI agents can interact with features

## Technical Implementation

### Architecture

The demo follows a clean architecture pattern divided into several key components:

1. **Core Plugin** (`wp-feature-api-demo.php`) - Initializes the plugin and loads dependencies
2. **Feature Registration** (`RegisterFeatures.php`) - Contains examples of feature registration
3. **Chat Controller** (`ChatController.php`) - Handles the REST API communication for the conversational interface
4. **Agent Implementation** (`Agent/BasicAgent.php`) - Processes natural language requests and maps them to features
5. **Client-Side Features** (`src/features`) - JavaScript-based features that run in the browser
6. **React UI** (`src/chat-app.tsx`, `src/components`) - Provides the conversational interface in WordPress admin

### Server-Side Features

The demo registers several PHP-based features:

```php
wp_register_feature(
    array(
        'id'          => 'demo/site-info',
        'name'        => __( 'Site Information', 'wp-feature-api-demo' ),
        'description' => __( 'Get basic information about the WordPress site.', 'wp-feature-api-demo' ),
        'type'        => WP_Feature::TYPE_RESOURCE,
        'categories'  => array( 'demo', 'site', 'information' ),
        'callback'    => [ $this, 'site_info_callback' ],
    )
);
```

These features demonstrate:
- Resource features (read-only data retrieval)
- Tool features (performing actions with side effects)
- Conditional availability based on installed plugins
- Category organization for feature discovery

### Client-Side Features

The demo also includes TypeScript/JavaScript features that run in the browser:

```typescript
export const logMessage: Feature = {
    id: 'demo/log-message',
    name: __( 'Log Message to Console' ),
    description: __( 'Logs a specified message to the browser console.' ),
    type: 'tool',
    location: 'client',
    categories: [ 'demo', 'logging' ],
    input_schema: {
        type: 'object',
        properties: {
            message: {
                type: 'string',
                description: __( 'The message to log.' ),
            },
        },
        required: [ 'message' ],
    },
    callback: ( args: { message: string } ) => {
        console.log( `[Demo Feature] ${ args.message }` );
        return { success: true, logged: true };
    },
};
```

These features demonstrate:
- Client-side execution of JavaScript
- Schema validation for inputs
- Integration with the WordPress client feature SDK

### Chat Interface

The demo includes a conversational interface implemented as a React application that:

1. Allows users to send natural language messages
2. Routes messages to a simple agent implementation
3. Maps user intent to available features
4. Executes features and returns results
5. Handles client-side feature execution via tool calls
6. Maintains conversation context

This interface demonstrates how AI agents (like ChatGPT or Claude) can discover and use WordPress features through the API.

## Key Ideas and Patterns

### Capability Discovery

The Feature API enables programmatic discovery of capabilities. This is particularly useful for:
- AI assistants that need to know what actions are possible
- Plugin developers extending WordPress functionality
- Headless CMS implementations that need to discover available endpoints

### Schema-Driven Development

The demo emphasizes the importance of well-defined schemas:
- Input validation using JSON Schema
- Clear output structure for predictable results
- Self-documenting capabilities through schema descriptions

### Cross-Environment Communication

The demo showcases how Features can be registered and executed across environments:
- Server-side PHP features
- Client-side JavaScript features
- REST API as the communication channel

### Agent-Feature Interaction Pattern

The demo implements a pattern where:
1. A user requests an action in natural language
2. An agent interprets the request
3. The agent discovers relevant features
4. The agent executes the selected feature
5. Results are returned to the user

This pattern is extensible to more sophisticated agents and features.

## Installation and Usage

1. Ensure the main WordPress Feature API plugin is installed and activated
2. The demo is loaded automatically by default (controlled by `WP_FEATURE_API_LOAD_DEMO` constant)
3. An admin notice confirms when the demo is active
4. Navigate to "Tools" > "Feature API Demo" to access the chat interface
5. Try natural language requests like "Tell me about this site" or "What plugins are active?"

## Testing via REST API

You can interact with features directly through the WordPress REST API:

```
# List all features
GET: /wp-json/wp/v2/features

# Execute a feature
POST: /wp-json/wp/v2/features/[feature-id]/run
{
  "param1": "value1",
  "param2": "value2"
}
```

This enables programmatic interaction with all registered features.

## Extending the Demo

The demo plugin is designed to be extended:

1. Add new features to `RegisterFeatures.php`
2. Create client-side features in `src/features/`
3. Enhance the agent in `Agent/BasicAgent.php`
4. Modify the UI components in `src/components/`

Use the demo code as a template for your own implementation of the WordPress Feature API.
